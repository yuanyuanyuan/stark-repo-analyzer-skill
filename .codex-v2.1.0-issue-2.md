## What to build

为 external hypothesis 建立 Evidence-First 安全门，使外部生成内容即使看起来包含文件路径或行号，也不能自行进入关键单元覆盖、Evidence Matrix 正式证据或风险证据。

该切片同时把当前锚点检查从字符串格式提升到当前 checkout 中真实存在的文件和有效行范围。完成后，可以端到端演示外部假设被拒绝、有效本地锚点被接受、失败原因进入质量门报告。

## Acceptance criteria

- [ ] 质量门验证所有正式 file:line 锚点对应当前仓库中存在的文件和有效行范围。
- [ ] 来源为 `external_hypothesis` 的内容不能计入 coverage 分子。
- [ ] 来源为 `external_hypothesis` 的内容不能写入 Evidence Matrix 的 source evidence 或风险 evidence 并通过 gate。
- [ ] URL、自然语言引用、越界行号和不存在文件均产生明确、可操作的失败原因。
- [ ] 合法本地锚点和既有双硬条件行为保持兼容。
- [ ] 测试覆盖伪造路径、陈旧路径、合法路径、行范围边界和 external hypothesis 污染场景。
- [ ] 质量门报告清楚区分格式错误、路径错误、行范围错误和来源边界错误。
