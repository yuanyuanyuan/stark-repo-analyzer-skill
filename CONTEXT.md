# Stark Repo Analyzer Skill Context

本项目用于重新实现一个仓库分析技能。基线讨论以当前工作树为起点，参考仓库作为外部语料，改造前后的比较以可复现的项目能力和交付物为准。

## Language

**当前项目**:
指当前工作树中的 `stark-repo-analyzer-skill`，其初始基线不继承此前 Git 历史中的实现或行为。
_Avoid_: 历史版本、旧实现

**参考语料**:
指 `参考仓库源代码/repo-analyzer-master` 中用于指导重新实现的源文件、文档和元数据，不代表当前项目已经具备这些能力。
_Avoid_: 当前实现、已实现功能

**基线**:
指在开始重新实现前，对当前项目状态、目标能力契约和可重复验证方式的明确记录。
_Avoid_: 改造结果、最终实现

**参考输出基线**:
指使用参考语料中的技能，对固定的代表性项目执行后保存的原始分析报告、运行信息和结构化检查结果，用于比较重新实现后的输出能力。
_Avoid_: 绝对标准、当前实现结果

**Graphify 结构证据层**:
指每次仓库分析开始时强制生成并验证的知识图谱输入层，用于发现模块候选、依赖路径与跨模块线索；它不替代核心路径的源码阅读和架构判断。
_Avoid_: 最终报告来源、源码替代品、可选增强

**最小纵向链路**:
指 V1 首先证明可重复运行的端到端分析范围：目标仓库、有效 Graphify 图谱、规模扫描、一个核心模块的源码验证、最小报告与结构检查。
_Avoid_: 仅报告生成、仅图谱构建、完整功能集

**LLM 引擎就绪**:
指 Graphify 按其官方自动探测规则，从当前环境变量、凭据或受信任配置中选出一个可用的语义后端；本技能不固定供应商、密钥或探测优先级。
_Avoid_: DeepSeek 必需、技能自行选后端、默认写死的 API key

**Graphify 自动化执行器**:
指用于可重复分析的 Graphify headless CLI 入口；它负责执行提取并返回退出状态，交互式 `/graphify` 只用于人工探索。
_Avoid_: 交互式 skill 作为回归执行器、agent 临场拼装命令

**分析工作区**:
指一次分析专属的可写产物目录，包含 Graphify 图谱、缓存、草稿、metadata、日志和最终报告；被分析仓库是只读输入，不承载这些产物。
_Avoid_: 在目标仓库写 `graphify-out/`、共享的临时目录、未隔离缓存

**Graphify 暂时性失败**:
指可识别的网络超时、429 或 5xx 等可能自行恢复的 LLM 调用错误；它与配置、权限、空图及产物完整性错误不同，后者不应通过盲目重试处理。
_Avoid_: 所有 Graphify 失败、无边界重试、配置错误重试

**有效 Graphify 图谱**:
指在分析工作区中同时具备可解析 `graph.json` 和 `GRAPH_REPORT.md` 的非空结构证据；节点、边与来源引用完整，且来源受限于目标仓库。`click` 试点额外要求至少一条可定位的 `EXTRACTED` 边。
_Avoid_: 只存在输出文件、空图、无来源的图谱、只有节点的 click 试点图谱

**源码裁决**:
指图谱发现与已读源码冲突时，以源码及其可定位证据为最终依据；图谱关系按置信度保留为导航线索、待验证项或风险项。
_Avoid_: 由 Graphify 覆盖源码、将推断直接写成结论、静默忽略冲突

**Graphify 深度提取**:
指 Graphify headless CLI 的 `--mode deep` 参数，用于生成更丰富的结构与语义关系；它是 V1 的强制建图行为，不等同于产品分析档位。
_Avoid_: repo-analyzer deep、可选图谱增强、用户选择的报告深度

**standard 分析档位**:
指 V1 唯一交付的 repo-analyzer 源码阅读与报告覆盖范围；产品级 `deep` 分析档位不在 V1 实现或验收范围内。
_Avoid_: Graphify 深度提取、V1 可选的 quick/deep、未实现的产品模式

**参考流程保真**:
指 V1 以参考 `repo-analyzer` skill 的阶段、方法论、报告逻辑和协作规则为行为基线，只加入 Graphify 所必需的前置建图、证据导航和交叉验证联动。
_Avoid_: 重新发明分析流程、独立分析平台、用图谱替代原流程

**Graphify Bootstrap**:
指正式分析前由 Agent 运行的依赖诊断、非敏感安装/升级和复检阶段；Agent 可修改本机工具环境，但不能生成、读取或披露密钥及私有 provider 配置。
_Avoid_: 让用户自行排查安装、Agent 编造密钥、静默绕过 LLM 就绪检查

**doctor**:
指 V1 的确定性 Graphify 预检脚本，按稳定退出码报告 ready、Agent 可修复、等待用户配置或阻断状态；它不执行正式建图、不修改目标仓库，也不管理密钥。
_Avoid_: 正式分析执行器、安装脚本、密钥探测器、仅供人工阅读的检查清单

**Graphify 侧车**:
指由 doctor 与 Graphify 构成的低侵入增强层：doctor 在建图前后承担所有程序化检测，Graphify 生成供原 skill 消费的结构地图，但不改变参考流程的设计逻辑、阶段责任或最终裁决。
_Avoid_: 在原 skill 各阶段分散检测、重写参考工作流、以图谱驱动最终结论
