# Stark Repo Analyzer Skill 领域上下文

本项目用于重新实现一个仓库分析技能。基线讨论以当前工作树为起点，参考仓库作为外部语料，改造前后的比较以可复现的项目能力和交付物为准。

## 领域语言

**当前项目**:
指当前工作树中的 `stark-repo-analyzer-skill`，其初始基线不继承此前 Git 历史中的实现或行为。
_Avoid_: 历史版本、旧实现

**参考语料**:
指 `vendor/repo-analyzer-reference` 中用于指导重新实现的源文件、文档和元数据，不代表当前项目已经具备这些能力。
_Avoid_: 当前实现、已实现功能

**Skill 核心交付包**:
指所有安装方式都必须完整交付的产品内容，包括 Skill 主定义、运行所需 references 和 Graphify gate；Graphify CLI 是由 gate 检测的外部依赖，不属于该交付包。
_Avoid_: 仅复制 `SKILL.md`、把 Graphify CLI 打包进 Skill、依赖完整仓库 checkout 才可运行

**基线**:
指在开始重新实现前，对当前项目状态、目标能力契约和可重复验证方式的明确记录。
_Avoid_: 改造结果、最终实现

**参考输出基线**:
指使用参考语料中的技能，对固定的代表性项目执行后保存的原始分析报告、运行信息和结构化检查结果，用于比较重新实现后的输出能力。
_Avoid_: 绝对标准、当前实现结果

**Graphify 结构证据层**:
指 Graphify 增强流程开始时必须生成并验证的知识图谱输入层，只用于发现模块候选、依赖路径与跨模块线索，为后续源码分析提供寻路指引；`standard` 与 `deep` 使用相同的 code-only 提取、本地 `cluster-only --no-label --no-viz` 和健康验证。Agent 可以在阅读源码后否决图谱候选，并负责确定最终模块边界。任何进入最终报告的实质结论仍必须回到源码阅读验证；已开始建图后，图谱缺失、为空或来源不可定位必须终止增强流程，不能静默忽略失败。
_Avoid_: 最终报告来源、源码替代品、自动裁决引擎

**Graphify 增强流程**:
指本机已有兼容 Graphify 时采用的默认分析流程；Graphify 图谱是该流程的强制前置输入，建图或健康验证失败必须终止。
_Avoid_: 原版兼容流程、Graphify 可用时跳过建图

**原版兼容流程**:
指本机缺少兼容 Graphify、Agent 已提供安装指引而用户暂不安装时，继续执行参考 repo-analyzer 纯源码分析的回退流程；它保留 `standard`、`deep`、对应源码阅读覆盖率和交互规则，唯一能力差异是没有 Graphify 导航 map，最终结果必须明确标记本次没有 Graphify 导航增强。
_Avoid_: Graphify 建图失败后的静默降级、伪造图谱证据

**最小纵向链路**:
指 V1 首先证明可重复运行的 Graphify 增强流程范围：目标仓库、有效 Graphify 图谱、规模扫描、一个核心模块的源码验证、最小报告与结构检查；原版兼容流程另行验证没有 Graphify 时的参考行为。
_Avoid_: 仅报告生成、仅图谱构建、完整功能集

**Graphify 自动化执行器**:
指用于可重复分析的 Graphify headless CLI 入口；它负责执行提取并返回退出状态，交互式 `/graphify` 只用于人工探索。
_Avoid_: 交互式 skill 作为回归执行器、agent 临场拼装命令

**分析工作区**:
指一次分析专属的可写产物目录，包含草稿、metadata、日志和最终报告；Graphify 增强流程还包含图谱及其导航证据。被分析仓库是只读输入，不承载这些产物。
_Avoid_: 在目标仓库写 `graphify-out/`、共享的临时目录、未隔离缓存

**用户分析运行**:
指用户触发 skill 对目标仓库执行的一次真实分析；最终只向用户交付一份 `ANALYSIS_REPORT.md`。Graphify 增强流程在工作区保留原始图/报告、规范化图/报告、导航 map、健康或失败摘要和正常分析草稿作为支撑证据，原版兼容流程则明确记录没有 Graphify 产物。
_Avoid_: 开发回归、UAT、fixture 合成运行

**开发验收运行**:
指维护 skill 时执行的单元测试、集成回归或真实 UAT；它可以使用额外的 manifest、哈希和测试状态，但这些不是用户分析运行的产品合同。大型图谱、缓存和完整转录保存在 Git 外，仓库只保留小型 fixture、结果摘要和必要失败证据。
_Avoid_: 用户分析运行、最终报告交付

**普通 UAT**:
指开发过程中针对模式、分支或输出合同进行的聚焦验收；它不要求完成整份仓库分析，也不能作为上线发布的真实回归通过证据。
_Avoid_: 真实回归 UAT、完整发布证据

**真实回归 UAT**:
指从用户等价入口完整执行到最终报告并保留可核验证据的端到端回归；它只在上线发布前或用户明确要求时执行。
_Avoid_: 静态 fixture、内部控制面直调、中断运行、普通 UAT

**有效 Graphify 图谱**:
指原始 Graphify 产物经过来源边界和位置规范化后，仍包含非空、来源可定位的节点与关系，并具备可解析的 `graph.json` 和 `GRAPH_REPORT.md`。原始产物中的少量越界、缺失或不可定位记录可以被过滤并记录，不因单条噪声否定其余可用导航证据。
_Avoid_: 只存在输出文件、规范化后为空、没有可定位关系的图谱、要求原始图零噪声

**源码裁决**:
指图谱发现与已读源码冲突时，以源码及其可定位证据为最终依据；图谱关系按置信度保留为导航线索、待验证项或风险项。
_Avoid_: 由 Graphify 覆盖源码、将推断直接写成结论、静默忽略冲突

**standard 分析档位**:
指 repo-analyzer 默认的常规源码阅读与报告覆盖范围，核心模块最低源码阅读覆盖率为 60%，次要模块为 30%；用户未明确指定模式时自动使用此档位，并在输入没有实质歧义时一键执行，不要求关注点或报告提纲确认。
_Avoid_: quick、Graphify 提取模式

**deep 分析档位**:
指 repo-analyzer 在 `standard` 基础上扩大源码阅读范围和验证深度的交互式分析档位，核心模块最低源码阅读覆盖率为 90%，次要模块为 60%；仅在用户明确要求深度分析时使用。Graphify 增强流程在建图和项目快速扫描后、原版兼容流程在项目快速扫描后进行一轮基于证据的集中询问，其中必须展示建议分析范围及选择理由供用户确认或调整；用户回答后直接继续，不再等待报告提纲确认。它不代表 Graphify 的提取模式。
_Avoid_: quick、Graphify deep、semantic extraction

**有界分析范围**:
指超大型仓库中为保证分析可执行而明确选择的模块、代码目录或文件集合，分析档位覆盖率只在该集合内计算。报告必须列出纳入内容、选择理由和排除范围，不得把范围内覆盖率表述成整个仓库覆盖率。
_Avoid_: 隐式抽样、任意挑选文件、全仓覆盖声明

**参考流程保真**:
指 V1 以参考 `repo-analyzer` skill 的外部调研、业务模块划分、并行源码分析、覆盖率记录、主 Agent 交叉验证和报告融合为行为基线，只加入 Graphify 导航增强，并采用已确认的 `standard`/`deep` 双模式与交互调整。
_Avoid_: 重新发明分析流程、独立分析平台、用图谱替代原流程

**subagent 能力降级**:
指运行环境不支持 subagent 时，在不降低覆盖率、源码验证和报告质量要求的前提下，改由主 Agent 顺序完成模块分析。Agent 必须先暂停、说明耗时影响，并由用户明确选择继续或停止，不能默认降级。
_Avoid_: 静默顺序执行、降低质量门槛、伪造并行记录

**Graphify 安装引导**:
指 Agent 检测到 Graphify 缺失或版本不兼容后，向用户提供官方安装或升级步骤，并让用户明确选择完成安装后复检，或仅本次进入原版兼容流程；Agent 不代替用户执行安装或修改本机工具环境，也不自动选择回退。
_Avoid_: Agent 自动安装、自动回退、无指引直接失败、声称用户已经安装

**Graphify 侧车**:
指围绕 Graphify 增强流程构成的低侵入程序层，只负责版本检查、建图、健康验证、来源路径规范化和导航 map；模式交互、模块划分、源码阅读、覆盖率、交叉验证和最终报告仍由 Agent 负责。
_Avoid_: 独立分析控制面、重写参考工作流、以图谱驱动最终结论

**Delivery Task（交付任务）**:
指会改变仓库正式文件、外部状态，或产生需保留并被后续依赖的计划/报告/审查结论的工作。纯讨论、只读查询和不留正式交付物的临时探索不是 Delivery Task。
_Avoid_: 把只读问答当成交付、把未写入仓库的闲聊结论当成已交付

**默认 Judge 交付任务**:
指除纯讨论、只读查询和不留正式交付物的临时探索外，默认必须经过独立 Judge 的 Delivery Task。仅当用户在当前任务中明确书面豁免时，才可省略 Judge；豁免必须记录豁免人、豁免范围和剩余风险。
_Avoid_: Agent 因改动小自行豁免、把历史豁免泛化到新任务、未记录范围的口头省略

**GitHub 源码 Release**:
指在本仓库远端创建带版本标签和发布说明的公开 GitHub Release；它证明该版本源码可被获取，不自动证明外部 marketplace 安装成功或 analyzer 真实回归 UAT 通过。
_Avoid_: marketplace 上架、已完成真实回归、仅推送分支

**公开分发身份**:
指用户安装时使用的 GitHub `owner/repo`、插件 marketplace 来源与仓库链接所共同表达的唯一分发地址。本项目当前采用 `yuanyuanyuan/stark-repo-analyzer-skill`；Skill 的稳定调用名仍为 `repo-analyzer`。
_Avoid_: 在 README、manifest 与远端之间保留多个分发地址、把分发仓库名误当成稳定 Skill 调用名

**参考项目致谢**:
指在公开入口中链接 `yzddmr6/repo-analyzer`，说明其作为 MIT 许可的参考语料对本项目的影响；致谢不表示当前项目是上游镜像、官方关联项目，或继承其支持范围。
_Avoid_: 隐去来源、暗示官方背书、把参考项目的兼容承诺转述为当前承诺

**Task Quality Gates（任务质量门）**:
指按风险触发的目标关、启动关、执行关与收尾关检查；分为轻量门与完整门。完整门字段写入 exec plan/progress，不另建并行状态机。
_Avoid_: 按改动文件数机械触发、用质量门替代 roadmap/spec

**Worker**:
指负责实现、自验并修改正式文件的执行角色。
_Avoid_: 自审冒充独立审查、越权改用户无关改动

**Judge**:
指独立、只读的审查角色；输出 `pass | revise | blocked`，至少做一项与核心风险相关的独立验证，不得修改正式树或放宽 UAT 证据上限。
_Avoid_: 顺手修代码、用偏好扩展需求、把静态扫描写成真实 UAT 通过

**Judge 审查包**:
指由 Orchestrator 在启动 Judge 前提供的、限定审查边界的输入：任务目标与验收项、启动基线、本任务拥有的文件范围、明确排除的用户已有改动、Worker 已执行与未执行的验证，以及阻塞级别规则。缺少任一项时 Judge 必须返回 `blocked: insufficient review package`，不得扩大扫描范围；范围外发现仅可作为非阻塞建议。
_Avoid_: 只给 plan/progress 后要求 Judge 猜测增量、用整棵脏工作树代替任务范围、将范围外意见作为阻塞项

**审查包生成**:
指 Orchestrator 在 Worker 启动时自动捕获工作区基线和任务文件边界，并在发起 Judge 时生成审查包；Worker 只补充任务目标、验收项与实际验证，用户不承担字段填写责任。
_Avoid_: 要求用户复制 prompt 或填写审查元数据、由 Judge 从脏工作区反推边界、Worker 事后伪造启动基线

**Judge 运行配置**:
指独立 Judge 使用固定的 `gpt-5.6-terra` 模型与 `medium` 推理等级，不因任务风险自动升档；每份 Judge Review 必须记录实际模型和推理等级。
_Avoid_: 依赖可漂移的用户默认值、按文件数量升档、未记录实际运行配置

**Judge 启动时点**:
指 Worker 完成自验并声明准备收口时，Orchestrator 为一个 Delivery Task 启动一次独立 Judge。hook 只在 `awaiting-judge` 时提醒调度，不在每次编辑或回合停止时重复启动审查。
_Avoid_: 审查中间实现状态、每次 hook 触发都创建新 Judge、遗漏收口前的独立审查

**Judge 迭代上限**:
指 Judge `revise` 后，Worker 修复并基于新的审查包重审的最大次数为三轮。同一 major 问题连续两轮未解、Judge `blocked` 或第三轮仍未通过时，停止自动迭代并交由用户决定。
_Avoid_: 无限重审、以轻微风格意见阻塞多轮、在未更新审查包时复用旧结论

**Judge 阻塞标准**:
指只有违反用户明确目标、验收项或审查包范围，可复现的功能错误、数据/安全风险、控制面状态错误，任务要求但缺失的验证证据，以及审查包缺字段或边界无法确认，才可导致 `revise` 或 `blocked`。风格偏好、可选重构、范围外发现和不影响当前验收的历史问题只能作为非阻塞建议。
_Avoid_: 将个人偏好写成阻塞项、借审查扩大需求、以范围外历史问题阻止当前任务通过

**Orchestrator**:
指在复杂或并发交付中分派 Worker/Judge、汇总结论并控制迭代轮次的调度角色。
_Avoid_: 用调度身份绕过 Judge 只读约束

**Blindspot Pass（盲点扫描）**:
指启动关中对可能遗漏的利益相关方、状态、失败路径与约束的显式列举。
_Avoid_: 事后回填假装事前已知

**Deviations（执行偏离）**:
指执行中相对原计划的实际偏离记录，含原因、取舍、影响与下次应提前确认项；无偏离时显式写“无”。
_Avoid_: 把未声明的范围扩大当成默认权力

**Boundary Check（边界检查）**:
指收尾关中对结果适用范围、失效条件、降级或回退方案以及验证证据的说明。
_Avoid_: 无边界的“已完成”宣告

## 主线总结

本文件是领域词汇真源：讨论产品行为时优先使用这里的名称和边界。最关键的区分是，Graphify 只提供经过健康检查的结构导航，源码负责最终裁决；用户分析、开发验收、普通 UAT 和真实回归 UAT 也是四类不同活动，不能互相冒充证据。协作侧的 Delivery Task、质量门与 Worker/Judge 只约束开发交付过程，不改变上述产品证据边界。
