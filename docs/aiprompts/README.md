# AI Prompt 目录

本目录存放可复用的 Agent 操作指引。直觉上，它像“特定场景的操作卡片”：帮助 Agent 重复执行一类工作，但不能决定产品行为或降低验收标准。

准确地说，Prompt 只提供操作指导，不具备仓库级权威；它必须服从 `AGENTS.md`、活动 roadmap、活动执行计划、产品合同和 `docs/dev-rules/`。类比的边界也在这里：操作卡片可以告诉你怎么做，却不能修改规则本身。

## 适合放入的内容

- 可重复使用的分析或审查 Prompt；
- 角色明确、边界有限的 Agent 任务说明；
- 可复用的交接或证据收集模板；
- 帮助 Agent 应用既有合同的示例。

每份 Prompt 必须写明用途、适用场景、必需输入、预期输出，以及它所依赖的合同或计划。

## 不应放入的内容

- 仓库级硬规则；
- 产品行为或输出合同；
- 验收标准或测试完成声明；
- 活动任务状态、密钥、凭据或机器私有配置。

如果 Prompt 与 `AGENTS.md`、活动 roadmap、活动执行计划、`docs/dev-rules/` 或产品合同冲突，必须修正或废弃 Prompt，不能反过来修改权威边界。

文件名使用 `{场景名}-prompt.md`。

## 主线总结

这里存“怎么重复执行某个场景”，不存“产品必须怎样工作”。Prompt 能辅助执行，但不能成为新的合同或验收入口。
