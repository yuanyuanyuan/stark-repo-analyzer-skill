# 用双模式真实 UAT 取代六仓库完整回归

> 此决定已被 [ADR-0022](0022-separate-focused-uat-from-real-regression-uat.md) 取代。以下正文保留历史原貌。

V1 的端到端验收只强制执行两个真实用户入口：一个小型仓库验证默认 `standard` 的一键运行，一个中型仓库验证 `deep` 的单轮集中询问与更高源码阅读覆盖率。Graphify 的空图、噪声过滤、越界来源、路径规范化和失败阻断等确定性边界使用小型 fixture 测试，不再以六个仓库的完整分析和大型运行档案作为 V1 完成条件。

## 影响

- 两个真实 UAT 都必须从用户等价的 skill 输入启动，不能用内部控制面直调或静态报告替代。
- `standard` 和 `deep` 的公开模式合同必须分别得到端到端验证。
- 六仓库历史基线可以作为非阻塞参考，但不再决定发布是否通过，也不需要继续提交新的完整运行产物。
