# 大型仓库允许显式有界分析

超大型仓库可以先声明本次分析的模块、代码目录或文件集合，再在该范围内计算 `standard` 或 `deep` 的源码阅读覆盖率。范围选择必须结合项目核心数据流、Graphify 候选、代码规模和用户关注点，并在报告中逐项列出分析了什么、为什么选择这些代码，以及哪些内容被排除；范围内覆盖率不得伪装成整个仓库覆盖率。

## 影响

- 覆盖率分母只包含公开声明的范围，未纳入的文件不能计入已覆盖或已验证。
- 报告必须同时给出纳入清单、选择依据和排除清单，使读者能够判断范围偏差。
- Graphify 只帮助发现候选，最终范围仍由 Agent 在源码快速扫描后确定。
- `deep` 在唯一一轮集中询问中展示建议范围和理由供用户确认或调整；`standard` 不阻塞询问，但在最终报告中披露范围和理由。
