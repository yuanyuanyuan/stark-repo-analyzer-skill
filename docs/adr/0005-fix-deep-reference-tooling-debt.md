# deep 必须偿还 reference Tooling Debt，禁止靠放宽门禁或永久降级收工

deep 在 TS/JS 上 reference-quality 失败的根因是分析器 Tooling Debt（例如 Graphify EXTRACTED 边未接线到 units `refs_status`，ctags 无可用 reference roles）。本阶段必须修复接线或接入可审计的 reference 提供方，使 deep 在能力真实具备时能 Full Delivery。

禁止：为变绿而放宽 `maxCoreIncompleteReferenceRate` 却不提升真实引用证据；禁止把长期 Degraded Delivery 当作 deep 成功终点。Degraded Delivery 仅是 gate 红时对 Primary Reader 的过渡交付；UAT「产品分析完整通过」仍要求 Full Delivery。在债未还清前，不得对外宣称 deep 完整通过。
