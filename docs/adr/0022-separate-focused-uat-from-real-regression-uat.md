# 区分普通 UAT 与真实回归 UAT

开发期间使用普通 UAT 聚焦验证模式选择、Graphify 安装分支、健康门和输出合同，可以在目标行为得到证明后停止，不要求完成整份仓库分析。只有上线发布前或用户明确要求时，才执行真实回归 UAT；真实回归必须从用户等价入口完整运行到最终报告，并保留可核验的输入、过程、退出状态、产物和失败证据。

## 取代关系

本决定取代 ADR-0020 中把固定数量的完整真实 UAT 作为日常 V1 完成条件的要求；代表性分支仍须验收，但普通 UAT 与发布级真实回归不再混为一谈。

## 影响

- 普通 UAT 的通过只能证明被检查的局部行为，不能宣称完整产品回归通过。
- 真实回归 UAT 不得由静态 fixture、内部控制面直调、手写报告或中断运行代替。
- 上线发布前完整执行 Graphify 增强 `standard`、Graphify 增强 `deep` 和无 Graphify 的 `standard` 兼容流程；无 Graphify 的 `deep` 组合由普通 UAT 验证，不再额外执行第四次完整分析。
- 真实回归的大型原始证据遵循 ADR-0019，保存在 Git 外；仓库保留轻量摘要和可核验定位信息。
