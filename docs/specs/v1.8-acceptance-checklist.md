# v1.8 人工验收清单（Soft Quality Gate）

> 来源：`evidence-first-v1-roadmap/v1.8-soft-quality-gate.to-spec.md` 的 Testing Decisions。
> v1.8 不实现硬质量门、quality-gate-report JSON、自动解析 Evidence Matrix、LLM judge 自动评分或阻止最终报告生成的逻辑。本清单用于人工验收最终报告前是否生成 Soft Quality Gate 摘要，以及未通过项是否反射进最终报告。

## 验收前置

- 选一个**代表性仓库**（建议含 3 个以上核心模块，且存在至少一处可能无证据或证据不足的判断）。
- 使用包含 v1.8 的 repo-analyzer 跑一次标准分析。
- 保留本次分析工作目录中的 `drafts/08-quality-gate.md`、`drafts/06-module-*.md`、`drafts/07-cross-validation.md`、`drafts/08-insights.md` 和最终报告。

## 验收项

### 1. 生成质量门摘要 ✅/❌

- 打开 `drafts/08-quality-gate.md`。
- 确认文件存在并包含质量门摘要，而不是空文件或跳过说明。

### 2. 质量门覆盖全部必需检查项 ✅/❌

- 检查摘要是否覆盖以下全部检查项，每项标记 ✅ / ❌ / N/A：
  - Evidence Matrix 完整性
  - 源码锚点
  - 风险抽样
  - Unsupported Claims
  - 预算执行
  - Repo Map 使用情况
  - 关键单元覆盖（v1.9 试点项）
- 不要求 Markdown 排版逐字一致，只核对字段语义完整。

### 3. 质量门引用实际草稿和证据 ✅/❌

- 检查每个 ❌ 未通过项是否写明原因，并引用具体草稿位置或证据缺口，而不是空泛打勾。
- 检查每个 ❌ 未通过项是否指出后续处理位置（进入最终报告的哪个区域）。

### 4. 未通过项进入最终报告限制说明或开放问题 ✅/❌

- 打开最终报告 `ANALYSIS_REPORT.md`。
- 对照 `drafts/08-quality-gate.md` 中的 ❌ 未通过项，检查每项是否出现在报告的限制说明、开放问题或 unsupported area 区域。
- 未通过项不得被静默忽略，也不得被写成确定性结论。

### 5. 质量门不阻塞最终报告生成 ✅/❌

- 确认本次分析完成并输出了最终报告，即使存在 ❌ 未通过项。
- 确认本次流程没有引入 JSON gate、gate CLI、LLM judge 自动评分或硬阻塞逻辑。
- v1.8 继续继承 v1.1 源码锚点、v1.2 Evidence Plan、v1.3 Evidence Matrix、v1.4 Unsupported Claims、v1.5 Risk Sampling、v1.6 Budget Profiles 与 v1.7 Markdown Repo Map。

```text
仓库：{repo}
模式：{mode}

项1 质量门摘要已生成：                    ✅/❌

项3 引用实际草稿和证据：                  ✅/❌
项4 未通过项进入最终报告限制说明/开放问题：✅/❌
项5 未越界引入硬阻塞/gate CLI/LLM judge：✅/❌
备注：{说明}
```

验收通过条件：项 1-5 全部 ✅。
