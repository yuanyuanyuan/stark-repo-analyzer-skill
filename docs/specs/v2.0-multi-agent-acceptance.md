# v2.0 多子代理验收规则

更新时间：2026-07-11  
关联：GitHub Issue #12 · ticket `docs/agents/tickets/issue-12.md`  
证据样例：`测试证据/v2.0/`

## 目的

把「多子代理执行」与「CLI/gate 机械链路」分开验收。`quality-gate-report.json` 中的 `allowed_to_synthesize:true` **只**表示确定性门控通过，**不能**单独证明 standard/deep 已完成真实多子代理模块分析。

## 判定矩阵

| 模式 | Evidence Plan 记录 | 多子代理验收 | 可声称的结论 |
|---|---|---|---|
| quick | 允许 `parallelism: degraded` + 串行顺序 | 不要求多子代理 | 串行 quick 链路；须标明 degraded |
| standard / deep | `parallelism: active` + 分工/产物/融合 | 必须通过 | 完整多子代理验收通过（仍须其它 gate 全绿） |
| standard / deep | `parallelism: degraded` 或缺少 active/分工/产物/融合 | **未通过** | 至多「CLI/gate 机械链路」或「部分通过」；**禁止**写完整通过 |

## 完整通过（standard/deep）最低记录

`evidence-plan.md` 必须同时满足：

1. 显式 `parallelism: active`（不得与 `degraded` 并存充数）
2. **实际子代理分工**（谁负责哪些模块/问题边界）
3. **每个子代理产物**（可追溯路径，如 `module-evidence/{module}.json`）
4. **主 agent 融合过程**（如何 merge 进 Matrix / `report.md` / 最终报告）

运行时支持 subagent 却仍写 `parallelism: degraded`：只能记部分通过/未完成，不得按完整通过关闭验收。

## Gate 合同（机器检查）

实现：`src/gate.js` → `parallelismExecutionCheck`

- **不得**仅检查 `parallelism` 字段是否出现
- quick：parallelism-execution 检查通过（仍受其它质量门约束）
- standard/deep：
  - `parallelism: degraded` → fail
  - 缺少 `parallelism: active` → fail
  - 缺少分工 / 产物 / 融合记录 → fail
- `allowed_to_synthesize` 为全部 checks 的合取；parallelism 失败则不得合成最终报告

自动化覆盖：`test/gate.test.js`（degraded 拒绝、active 缺失拒绝；quick 允许 degraded）

## 文档与证据口径

验收报告与对比报告必须：

- 总判定区分：自动化基线 / doctor-scan-units-gate 机械链路 / 报告质量 / **多子代理执行**
- 当三模式 Evidence Plan 均为 degraded 时，写明「没有多个子代理参与」
- 不得用历史 `allowed_to_synthesize:true` 顶替当前 gate 或当前多子代理结论

权威文案位置：

- `测试证据/v2.0/ACCEPTANCE_RESULT.md`（含「多子代理验收规则」节）
- `测试证据/v2.0/RUN_LOG.md` § Issue #12 复核
- `测试证据/v2.0/COMPARISON_REPORT.md`
- `skills/repo-analyzer/SKILL.md` Phase 5–6
- `skills/repo-analyzer/references/evidence-first-v2.md` Evidence Plan 模板

## 恢复 v2.0「完整通过」的条件（可选升级，非默认 PR 硬条件）

1. 至少对 standard 或 deep 重跑一次**真实多子代理**分析
2. `module-evidence/*.json` 与最终报告吸收各子代理产物
3. Evidence Plan 满足上文四要素且 gate 全绿（含 parse/reference/report-depth 等其它门）
4. 更新 ACCEPTANCE_RESULT / RUN_LOG / COMPARISON_REPORT 的判定与证据路径

## 反例（禁止）

- 仅因 `parallelism` 字符串存在就判多子代理通过
- 将 `parallelism: degraded` 等价为「质量门已通过故验收完整通过」
- 无子代理分工/产物/融合记录却在对外材料写「v2.0 完整通过」
