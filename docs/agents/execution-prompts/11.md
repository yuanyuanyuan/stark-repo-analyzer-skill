- 正式 PR：Unit 节合格 +（E2E 合格或 E2E N/A+替代合格）+ UAT 节 pass 且步骤具体
- draft（UAT blocked/failed 或 allow_failed_pr）：仍须填齐已有部分的**命令/摘录/失败细节**，并标明缺口
- 证据不合格 → G3 **不得**选「正式可合并」；应补跑补写 body 或停

#### 写入方式
- `gh pr create --title "..." --body "$(cat <<'EOF' ... EOF)"` 或等价；更新时用 `gh pr edit --body`
- Loop 小修 push 后：**重跑验证并改写** Test Evidence 各节，删除过期结论

### 3.8.2 G3 检查清单（给人看）
展示给用户确认前，执行器应自检 PR body 是否含 Unit/E2E/UAT 三节且每节含 command + excerpt（或 N/A 替代细节）。缺则先补 body 再弹 G3。

### 3.9 G2 否决
硬停；不改 tickets；不建 branch。

## 4. 强制 Loop 轮
- 触发：PR 已开出（正式或 draft）后 **自动**进入
- 必做：对照 PR/测试/UAT 写差距；更新 memory lessons+INDEX；**回写 planning**（P6：差距、Done/Pause/Next、task_plan 终态）；Next LOOP packet 或 Done/Pause
- 无需改代码：不弹人，直接收束（progress 标 complete / pause）
- 需要同 PR 小修：先确认 修 / 只记教训不修 / 停
  - 禁止自动小修 push；禁止新 scope/新票/第二 PR/大重构
- 若确认「修」：
  - 同 branch 改代码；planning 回到对应 phase（通常 P3/P4），**禁止**假装 P6 已结束
  - **完整**重走 Verification + UAT + **G3**
  - 更新 **同一 PR** body 与状态（**按 3.8.1 刷新详细 Test Evidence 各节**）；不关旧开新
  - 每步同步 progress.md
- 主路径硬停未开 PR：Loop = 失败复盘，写 memory + **planning（失败阶段/blocker/下一步）** + 报告

## 6. 完成定义（本流水线 userGoalDone）
以下之一：
- 正式 PR 已开/已更新，**PR body 含 3.8.1 合格详细测试证据**，Loop 轮已完成（含 memory），**且** planning 三文件反映终态（P6 complete 或明确 Pause + 下一步）
- draft PR 已开/已更新，正文诚实标注状态且**已有部分证据仍具体（命令/摘录/失败细节）**，Loop 轮已完成，**且** planning 终态诚实（含 draft/blocked 原因）
