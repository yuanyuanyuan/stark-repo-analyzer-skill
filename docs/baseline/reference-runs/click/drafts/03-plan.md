## 模式与门槛

本次按用户确认使用 `standard`：核心模块目标至少 60%，次要模块目标至少 30%。覆盖率口径沿用参考 skill：实际通过读取命令请求过的行范围并集 / 模块文件总行数。为便于复核，本次对前三个核心模块和 completion、testing 文件执行了完整文件读取；平台模块执行了所有文件的关键实现区段读取，聚合覆盖率高于 30%。

## 覆盖计划

核心模块优先覆盖入口、核心数据结构、解析/分发流程、资源生命周期和帮助输出；每个核心草稿必须回答角色、问题、方案权衡、数据结构、流程图和模块协作。

次要模块采用批量说明：描述职责、整体角色、实现方式和特殊边界。不能从当前 commit 证实的动机写为“待验证”。
