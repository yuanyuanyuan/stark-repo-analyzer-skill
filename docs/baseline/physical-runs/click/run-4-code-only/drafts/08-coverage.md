# 覆盖率汇总

口径：实际读取的源码行范围并集 / 纳入模块分母的有效源码行数。该指标是静态阅读覆盖率，
不是 pytest、coverage 或运行时行为覆盖率。`core.py` 在命令模型和参数模块之间有重叠，
总体核心合计按去重后的模块责任范围计算。

| 模块 | 类型 | 文件数 | 有效代码行 | 已读行数 | 覆盖率 | standard 门槛 | 达标 |
|---|---|---:|---:|---:|---:|---:|---|
| 命令模型与上下文执行 | 核心 | 2 | 4350 | 4350 | 100% | 60% | ✅ |
| 参数解析与类型转换（去重增量） | 核心 | 2 | 1908 | 1908 | 100% | 60% | ✅ |
| 终端交互与帮助输出 | 核心 | 3 | 1926 | 1926 | 100% | 60% | ✅ |
| Shell completion、testing、平台/异常/API | 次要 | 10 | 4104 | 3189 | 77.7% | 30% | ✅ |

## 总体

- **合计：** 核心与次要模块均通过 standard 覆盖率门槛。
- 核心模块：`8,184 / 8,184 = 100%`，高于 standard 60% 门槛。
- 次要模块：`3,189 / 4,104 = 77.7%`，高于 standard 30% 门槛。
- `src/click` 有效代码总量约 `12,288` 行；由于模块边界存在共享文件，不能把各表简单
  相加后当作全仓库覆盖率。

## 未覆盖与验证边界

次要模块的未读部分主要是 `_compat.py`、`_termui_impl.py` 等平台分支；它们不影响对
Linux/非 TTY 主要路径的静态理解，但不支持 Windows 行为已验证的结论。测试执行、真实
shell 进程、pager/editor 和跨平台行为均未纳入本指标。
