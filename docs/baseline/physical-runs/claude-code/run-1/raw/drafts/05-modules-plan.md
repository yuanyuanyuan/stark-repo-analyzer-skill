# 模块与叙事计划

## 叙事主线

`启动与命令边界` →[决定会话形态并构造运行上下文]→ `交互会话调度` →[将模型请求转化为能力调用]→ `工具与 MCP 能力面`

这条主线刻意不按目录排列：读者先理解“如何进来”，再理解“一回合如何运转”，最后理解“回合在何处获得外部行动能力”。

## 报告结构

1. 范围、证据与限制
2. 业务问题和项目全景
3. 总体架构与主数据流
4. 启动与命令边界
5. 交互会话调度
6. 工具与 MCP 能力面
7. 跨模块验证、风险和重设计建议
8. 综合评价与启发

## 非核心结构扫描

`services/analytics`、memory、compact、remote、plugins、LSP、voice、Ink UI 和大量 `commands/` 子目录属于本次结构扫描覆盖面。除非被核心调用链直接引用，不在标准模块深读覆盖率中计入；该选择是规模控制，不代表这些模块不重要。
