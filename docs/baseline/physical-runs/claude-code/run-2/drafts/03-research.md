# 研究草稿

## 项目解决的核心问题

从代码形态看，这是一个把 LLM 会话、工具调用、终端交互、远程会话和企业策略整合到单一 CLI 中的 agent runtime。用户输入进入 REPL 后，需要同时处理模型流式响应、工具批准、会话恢复、MCP 外部能力和多种远程传输。

## 竞品/同类项目对比

外部竞品调研：`not performed`。仅从源码可比较的技术路线是：本地 terminal agent、MCP client host、远程 worker/session client 被统一收敛在同一个进程，而不是拆成独立 daemon。

## 为什么需要单独做这个项目

单独的 API client 不能提供安全的工具批准、交互式终端渲染、插件/技能加载和会话持久化；单独的 shell wrapper 又难以维护模型流、工具状态和远程重连之间的一致性。源码显示项目价值在于把这些运行时边界放进一个可编排的 CLI。
