# 分析计划

## 模式与规模

- 请求模式：`standard`。
- 快速静态计数：3,274 个 Rust/TypeScript/JavaScript/Python 实现文件（命令见执行日志）。该计数不是有效代码行数，未剔除测试、生成物和供应商代码，故不把它误报为 LOC。
- 结论：这是超大型多 crate 仓库。采用**有界标准模式**：对选定核心模块力求达到 60% 文件行覆盖率，对选定次要模块达到 30%；全仓覆盖不作声称。
