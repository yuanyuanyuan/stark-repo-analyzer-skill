- [x] D04 确认第一条最小纵向链路
  - 依赖：D01-D03
  - 范围：本地 `click` 输入 -> Graphify 深度建图成功且图谱非空 -> 规模扫描 -> 一个核心模块源码验证 -> 最小报告 -> 图谱与结构验证。
  - 失败边界：doctor preflight/post-graph 非零时中止本次分析，不得降级。
  - 验证：用户已确认范围；决策记录于 `docs/adr/0003-graphify-is-a-required-structure-evidence-gate.md`。
  - 模式：V1 仅实现 repo-analyzer `standard`；Graphify `--mode deep` 是强制提取参数，不表示产品级 `deep` 分析。
- [ ] D05 用一个任务验证拆分粒度
  - 依赖：D04
  - 首个负向样本：当前仓库的 `graphify-out/` 报告为 0 nodes / 0 edges / 0 communities，验证器必须稳定拒绝。
  - LLM 闸门：doctor 必须调用 Graphify 的自动探测逻辑，确认实际 backend/model；不允许技能写死 DeepSeek 或自行确定探测优先级。
  - Bootstrap：Graphify 不可用时，Agent 可执行官方非敏感安装/升级和复检；API key、企业代理、私有 provider 地址必须由用户提供。
  - 执行器：使用 `graphify extract <target> --mode deep --out <WORK_DIR>`，不传 `--backend`；记录 Graphify 版本、退出码、实际 backend/model 和产物路径。
  - 输出边界：只接受 `$WORK_DIR/graphify-out/` 中的产物，目标仓库不得出现本次运行写入的 `graphify-out/`。
  - 自愈：仅网络超时、429、5xx 可退避重试，最多两次；无可用 LLM、SDK/权限/执行器错误、空图和无效产物立即失败。
  - 健康：doctor post-graph 检查 `graph.json`、`GRAPH_REPORT.md`、节点、边和来源引用；`click` 还必须有一条带来源定位的 `EXTRACTED` 边。
  - 证据：Graphify map 作为侧车上下文；源码裁决图谱冲突，`EXTRACTED` 验证核心路径，`INFERRED` 标待验证，`AMBIGUOUS` 只作为风险/疑问，并将冲突写入交叉验证草稿。
  - 验证：同一有效图谱任务连续两次可独立完成并通过验证，且空图或无可用 LLM 的任务稳定失败；否则继续拆细。
