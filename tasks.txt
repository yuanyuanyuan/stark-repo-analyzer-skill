- [x] D05 用一个任务验证拆分粒度
  - 依赖：D04
  - 首个负向样本：当前仓库的 `graphify-out/` 报告为 0 nodes / 0 edges / 0 communities，验证器必须稳定拒绝。
  - 模型边界：doctor 必须确认 code-only 模式和 semantic extraction disabled；backend/model 不参与当前路径，也不得被写入运行元数据。
  - Bootstrap：Graphify 不可用时，Agent 可执行官方非敏感安装/升级和复检；当前 code-only 路径不需要 API key、企业代理或私有 provider 地址。
  - 执行器：使用 `graphify extract <target> --code-only --no-cluster --out <WORK_DIR>`，随后使用 `graphify cluster-only <WORK_DIR> --no-label --no-viz` 生成 normalized 报告；记录 Graphify 版本、退出码、raw/normalized 产物路径和 doctor JSON。
  - 输出边界：只接受 `$WORK_DIR/graphify-out/` 中的产物，目标仓库不得出现本次运行写入的 `graphify-out/`。
  - 失败：code-only 是本地确定性路径；版本、权限、执行器、空图、无效产物和源边界错误立即失败，不重试或伪造成功。
  - 健康：doctor post-graph 检查 `graph.json`、`GRAPH_REPORT.md`、节点、边和来源引用；`click` 还必须有一条带来源定位的 `EXTRACTED` 边。
  - 证据：Graphify map 作为侧车上下文；源码裁决图谱冲突，`EXTRACTED` 验证核心路径，`INFERRED` 标待验证，`AMBIGUOUS` 只作为风险/疑问，并将冲突写入交叉验证草稿。
  - 验证：有效图谱必须通过 preflight/post-graph doctor；空图、缺失来源引用或目标仓库写入必须稳定阻断。
  - 当前证据：`acceptance/doctor-self-test.sh` 已证明两次独立健康图分类、空图稳定返回 30、工作区越界拒绝和无凭据输出；click 的两次独立 Graphify 归档运行位于 `docs/baseline/physical-runs/click/run-2` 和 `run-3`，`physical-repeatability-check.sh` 通过；真实 click/codex-plugin 控制面运行进一步通过 normalized graph 的 post-graph 健康门。
